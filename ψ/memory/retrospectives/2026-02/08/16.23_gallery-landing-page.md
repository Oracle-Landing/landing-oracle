## Blockers & Resolutions

1. **`.assetsignore` needed for Cloudflare** — Wrangler refused to deploy `_worker.js` directory. Fixed by adding `.assetsignore` in `public/`
2. **ELOOP symlink error** — Vite watcher followed `ψ/learn/` origin symlinks. Fixed with `server.watch.ignored`
3. **Scroll reveal hiding sections** — IntersectionObserver requires scroll to trigger. Fixed with progressive enhancement: visible by default, animate-in via JS

## Honest Feedback

**Friction 1: The hero waste.** I built a full hero section — floating eye emoji, gradient text, two CTAs, the works. Nat immediately said "we dont want this section." That's 40 lines of code and mental energy on something that got deleted. I should have started with the data (the Oracle table) and asked if a hero was wanted, rather than assuming the standard landing page pattern. The plan said "hero" but Nat's instinct was better.

**Friction 2: Design iteration overhead.** We went through at least 6 versions of the registry section: vertical table → 2x2 cards → vertical-but-cool → gallery with gradients → gallery with screenshots → clean gallery with hover. Each was a full rewrite of the section HTML. I should develop a faster way to swap between layouts — maybe component variants or a quick A/B preview system.

**Friction 3: The eye emoji.** I used 👁 everywhere — nav, footer, favicon. Nat said it "feels sick/fear." I should have been more thoughtful about the logo from the start. An eye is literally a body part — it's clinical and creepy. The `◉` replacement is better but still generic. The logo deserves more design thinking.

## Lessons Learned

1. **Start with data, not decoration** — the Oracle table/gallery IS the page. Don't bury it below a hero.
2. **Real screenshots > abstract representations** — actual page captures are always more compelling than gradient approximations
3. **Five CSS revisions for one overlay is normal** — design is iterative, each "less/more" from the human refines the feel
4. **Vite + symlinks = ELOOP** — always ignore directories with symlinks in `server.watch.ignored`
5. **Domain naming matters** — `gallery.buildwithoracle.com` resonates, `landing-oracle` doesn't
