## Honest Feedback

**Friction 1: Iteration without committing.** The current state has 4 modified files that represent a significant simplification (removing philosophy, masonry, filter, adding dot tooltips). But none of it is committed. If the session crashes or Nat switches context, this work could be confusing to pick up. The intermediate state between "masonry + filter" (committed) and "simple grid + dot tooltips" (uncommitted) is fragile. Should have committed the simplification as its own step.

**Friction 2: The 3-column detour.** Sessions 3 and 4 (35 minutes total) were spent trying to make a 3-column layout work — live in 2 columns, known in a scrollable third column. This was a plan-mode implementation that got reverted. The lesson: with only 4 live items, layout complexity doesn't help. Should have recognized earlier that the data doesn't support the layout. "Patterns Over Intentions" — look at what you have, not what you wish you had.

**Friction 3: Empty state was an afterthought.** The question "if no data how to show?" came from Nat, not from me. I should have raised this during the initial build. A gallery page without data handling is incomplete. The birth issue link is obvious in hindsight — it's the origin story, the natural thing to show when the gallery is empty. This should have been in the original plan.

## Lessons Learned

1. **Commit intermediate states** — When simplifying a feature (removing masonry, filter, philosophy), commit the simplification separately before moving on. Don't let 4 modified files linger.
2. **Empty state first** — For any data-driven page, design the empty state before the full state. What does the page say when it has nothing to show?
3. **Data dictates layout** — 4 items don't need masonry, columns, or filters. Simple 2x2 grid. When there are 20+ items, revisit.
4. **Dot grid is the real visualization** — The screenshot gallery shows the live sites, but the dot grid tells the whole story: 4 lit, 16 known, 56 dim. That's the gap.
5. **Birth issue as empty state** — When there's no data, link to where the Oracle was born (#154). The origin story fills the void.
